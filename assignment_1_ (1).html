Convention for file names

The reason file names are saved as they are so the web browser can parse the request correctly. Saving a file name such as this one as "assignment_1_IT130_html" will be able to be read by the browser, you cannot save the file name with normal spaces because _ is how the browser accepts spaces. Secondly, making sure ending your file name with ".html" so the web server knows the type of document to present

Web accounts on a web server

the reason a person would need an account on a web server to display their web pages. When a request is made through a web client the web server reads the request and sends HTTP response. Without having a account with a web server, an error would pop up to the person using the web client making the request.
